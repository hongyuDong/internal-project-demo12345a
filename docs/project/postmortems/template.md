## 3. 根因 (Root Cause)

### 直接原因
PR #1234 在 `auth_service.py` 改了 session validation，遗漏 null 检查。当 Redis session 过期返回 null 时，新代码抛 500。

### 根本原因（5 Whys）

1. **为什么 500？** 因为 Redis 返回 null 时没有 fallback
2. **为什么没 fallback？** 因为重构时漏掉这一支分支
3. **为什么漏掉？** 因为没写对应的单元测试
4. **为什么没测试？** 因为"重构不影响功能"的错误假设
5. **为什么没阻止？** 因为 CI 没强制要求回归测试

### 促成因素 (Contributing Factors)
- PR 描述不够详细（没说改了哪些边界场景）
- Reviewer 没注意 null 路径
- 集成测试没有覆盖 Redis 异常场景

## 填写指南

### 时间线
- **必须**精确到分钟
- 包含告警 + 决策 + 修复 + 验证
- 标注操作人（事后追责不追究，记录改进用）

### 根因
- **5 Whys** 必须挖到第 3 层以上
- 区分**直接原因** vs **根本原因** vs **促成因素**

### 影响
- **量化**：多少用户、多少钱、多少人小时
- **业务影响** > **技术影响**

### 行动项
- 每个行动项必须有 **Owner + Due Date**
- 必须**可执行**（不能是"提高意识"这种空话）
- 季度 review 跟踪进度

### 评审签字
- **不签字 = 没完成**
- 所有签字后才公示
