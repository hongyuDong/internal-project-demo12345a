# /track [可选: 任务 ID 或 archive]

查看进度并报告风险。

## 工作流

### 1. 调用 tracker sub-agent

让 tracker 执行检查。

### 2. 输出报告

```markdown
## 📊 跟踪报告: [PROJ-XXXX] <name>

### 当前状态
🟢 健康 / 🟡 落后 / 🔴 阻塞

### 进度
```
████████████████░░░░  65%
完成 13/20 subtasks
```

### 关键路径
- 阶段 1 ✅ 完成
- 阶段 2 ✅ 完成
- 阶段 3 🟡 进行中 (60%)
- 阶段 4 ⚪ 待开始

### 实际工时
- 预估: 5h
- 实际: 4.5h
- 偏差: -10% ✅

### 最近更新
- HH:MM — 完成 3.2 单元测试
- HH:MM — 完成 3.1 实施

### 风险
- 🟢 低 / 🟡 中 / 🔴 高

### 阻塞
- 无 / <描述>

### 预计完成
- 按计划: YYYY-MM-DD HH:MM
- 实际预测: YYYY-MM-DD HH:MM (提前 Xh)

### 下一步 subtask
**3.3**: <名称>
- 完成标准: ...
```

### 3. 如有异常，单独报告

#### 🔴 阻塞

```markdown
🚨 **阻塞预警**

**任务**: PROJ-XXXX
**当前 subtask**: X.Y
**卡了**: X 分钟

**可能原因**:
1. ...
2. ...

**已尝试**:
- ❌ ...

**建议**:
- 选项 A: ...
- 选项 B: ...

**需要决策**: <谁 + 决定什么>
```

#### ⚠️ 落后

```markdown
⚠️ **进度落后**

**计划**: X%
**实际**: Y%
**落后**: Z%

**根因**:
- ...

**建议**:
- 缩小范围
- 加人
- 延期
```

#### ⚠️ 风险升级

```markdown
⚠️ **风险升级**

**原等级**: 🟢
**新等级**: 🟡 / 🔴

**触发事件**: ...
**影响**: ...
**建议**: ...
```

## 选项

```bash
/track                       # 当前任务
/track PROJ-1001             # 指定任务（如有多个）
/track --archive             # 看归档任务统计
/track --health              # 健康度总览
```

## 反模式

❌ **不要**：
- 报喜不报忧
- 只看进度不看风险
- 不给行动项
- 等用户主动问才报告

✅ **应该**：
- 数据驱动（实际工时 / 进度百分比）
- 风险量化
- 给具体建议
- 主动预警
